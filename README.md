# Название проекта: RoleMaster

### Описание:
Данный репозиторий представляет собой сайт, который идет обновлением для консольной мини-игры. На сайте предоставляется возможность зарегистрировать новый аккаунт, что не было реализовано в консольной мини-игре, но теперь будет исправлено с этим обновлением, вход в свой аккаунт, где после успешного входа игрок будет отправлен на главную страницу, где он сможет найти обратную связь с разработчиком, немного информации о разработчике, а также ознакомиться с гидами, который есть в консольной мини-игре, на странице будет возможность перейти на страницу своего профиля или выйти из аккаунта. Для игроков с правами (Administrator, OWNER, GOD) будет реализована дополнительная кнопка на сайте, где будут отображены таблицы с данными из бд (в будущем их можно будет изменять).

### Установка и запуск:
1. Иметь уже заранее созданный проект - `Console-mini-game`
2. Скачать весь проект из этого репозитория
3. Убедиться, что существует бд (game_db_1) или та, что вы создали самостоятельно, а также в наличии таблиц и данных в них
4. Освободить перед запуском порт: 8080

### Пример использования:
1. Проверяем наличие всех файлов и таблиц с данными в базе данных
2. Проверяем свободен ли порт 8080. Команда для проверки - <br>
`netstat -ano | findstr 8080` <br>
3. Запускаем WebServiceApplication
4. Ищем в консоли вот эту строчку -> <br>
`Started WebServiceApplication in 11.203 seconds (process running for 11.91)` <br>
это значит, что все успешно запустилось и можно перейти по ссылке: http://localhost:8080/login.
5. Вводим в поля username и password данные из аккаунта в таблице users. Аккаунт выбирайте любой, который вы создали самостоятельно
6. Вход в случае, если осуществляется с правами обычного пользователя, то переходим по кнопке "Профиль", затем нажимаем кнопку "Выйти"

### Участники и их роли:
1. Тимлид - это руководитель команды, который отвечает за качественный и вовремя реализованный продукт, а также поиск каждому участнику команды задачи.
2. Java backend разработчик - программист, который занимается разработкой технической стороной сайта, работой с сервером и настройкой всех запросов к бд.
3. Дизайнер - отвечающий за внешний вид сайта
4. JavaScript разработчик - программист, который связывает fronted и backend, чтобы все запросы вызывались правильно и сайт исправно реагировал на действия пользователя
